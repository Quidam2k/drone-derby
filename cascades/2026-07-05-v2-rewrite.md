# Cascade: Drone Derby v2 — From-Scratch Rewrite

**Created:** 2026-07-05
**Status:** Phases 1–4 DONE — LIVE at https://drone-derby.pages.dev (prod push round-trip verified). Todd still owes: fun-playtest verdict + phone/iOS installed-PWA push check.

## Vision

Async multiplayer RoboRally-style programming game, browser-first PWA (no app
stores). Core loop: get a push notification → watch the last turn replay with
playback controls → program your 5 registers → submit → when the last player
submits, the turn executes and everyone gets notified. Games span days to a
week+.

## Cross-Phase Decisions (locked unless revisited)

- **Backend: Convex** (DB + auth + reactive queries + serverless functions).
  Replaces the entire old Express/Postgres/Redis/Socket.io stack. Pivot to
  Supabase possible later; keep Convex-specific code confined to `convex/`
  and thin client hooks.
- **Engine is a pure function**: `(gameState, programs, seed) → (newState,
  EventLog)`. Zero dependencies, deterministic (seeded RNG), lives in
  `src/engine/`, imported by BOTH the client (hot-seat, previews, replay) and
  Convex functions (authoritative execution). Never touches DB/DOM.
- **EventLog is the animation contract**: turn execution emits atomic events
  (move, rotate, push, conveyor, laser, damage, fall, respawn, checkpoint…)
  grouped by register. The replay player consumes only the EventLog. Stored
  per turn in Convex so any past turn is replayable.
- **Rendering: DOM/CSS grid** for the game board (tiles + transitions), NOT
  canvas. (Amended 2026-07-06, Todd: the level editor is ALSO DOM/CSS grid,
  reusing the game's Board/Tile renderer — WYSIWYG, zero new deps. Canvas is
  off the table unless a future feature demands it.)
- **State: Zustand** for client state. No Redux. No MUI — plain CSS
  (custom properties, grid/flex); can add Tailwind later if wanted.
- **Auth: Convex Auth** (magic link + Google OAuth). No passwords, ever.
- **One app, one package.json**: Vite + React + TS at repo root, `convex/` for
  backend functions. No monorepo/workspaces.
- **Old codebase archived** at `legacy/` — reference only (card/tile ideas,
  editor UX). Never import from it.
- **Rules scope (MVP)**: RoboRally-lite. Cards: Move 1/2/3, Back Up, Rotate
  L/R, U-Turn, priority 10–840. Tiles: floor, walls (edge-based), pits,
  conveyors (normal + express), gears, checkpoints, spawn docks, board
  lasers. Robots fire lasers. Damage locks registers (5+) and kills at 10;
  3 lives; respawn at last checkpoint. Cut for MVP: pushers, repair sites,
  option cards, power-down.

### Phase 5 — Async Quality-of-Life + Polish  [DONE 2026-07-06 (code + dev E2E)]
**Scope decision (Todd, 2026-07-06): NO turn deadlines, NO AFK enforcement.**
Play-when-you-can, like Words with Friends — deadlines deferred indefinitely
(may return later as a per-game *option*). Game is among friends, so social
pressure (nudges) replaces enforcement.
Delivered:
- **Nudge** (`games.nudge` mutation): any member not being waited on pokes
  the waiting players via the existing push pipeline; lobby variant (host →
  everyone else). Rate limit: one per game per 12h (`games.lastNudgeAt`;
  `nudgeAvailableAt` returned by the `game` query). UI: 👉 button in
  WaitingView's waiting note, disabled with next-available tooltip during
  cooldown. Server rejects bypass attempts with a clean error (verified via
  authed HTTP-API probe).
- **Speech bubbles** (Todd's 2026-07-06 request): optional ≤60-char line
  submitted with a program (`taunt-input` in ProgrammingView, both modes).
  Stored on submissions, folded into the turns row at execution
  (`turns.taunts` by player name), returned by `games.turn`. Rendered by
  ReplayPlayer: bubble pops at the player's first card-revealed event and
  lingers 8 events (`src/components/replay/taunts.ts`, unit-tested). Rides
  ALONGSIDE the EventLog — engine/event union untouched. Hot-seat threads
  taunts through the store (`pendingTaunts`).
- **History browser** (`src/components/online/HistoryBrowser.tsx`): "Watch
  past turns" from WaitingView + game-over screen; replays any executed turn
  via `games.turn` and NEVER calls markTurnSeen. Rendered before the
  unseen-turn catch-up branch so an in-progress browse isn't yanked.
- **Stats**: GameOverScreen per-player summary (checkpoints/lives from the
  final GameState, shared with hot-seat); lobby rows show `turn N` chip +
  per-player ✓/⏳ badges (myGames data, no new storage).
- Verified: typecheck + 61 tests green (3 new taunt-window tests); prod
  build; two-browser-context Playwright E2E on the dev deployment — nudge
  send + reactive cooldown + server-side rejection, bubbles on both screens
  turn 1, history list/replay/return with turn-3 programming state intact,
  lobby badges (screengrab/phase5-*.png).
Deferred to playtest verdict (gate still open): rules/pacing tweaks, mobile

Delivered:
- **`src/engine/validate.ts`** — pure `validateBoard(board)` →
  `{errors, warnings}`; errors: size 6–16, full grid, 2–4 spawns + ≥1
  checkpoint numbered 1..n contiguous, walls/lasers in bounds, no emitter
  on a pit. Warnings: bland board (no hazards), express belt not part of a
  belt line, spawn in a laser line. Reused server-side in 6b.
- **`src/store/editorStore.ts`** — draft board + tools (conveyor dir/express,
  gear spin; wall/laser place on cell EDGES, laser fires away from its

  auto-numbering (lowest free number; erasing renumbers nothing — validation
  flags gaps), resize preserving overlap, debounced localStorage draft
  (`droneDerby.editorDraft`), validation recomputed per change.
- **`src/components/editor/`** — EditorScreen at `#/editor` (works without
  Convex; lobby card links to it): toolbar (name, W/H steppers, undo/redo +
