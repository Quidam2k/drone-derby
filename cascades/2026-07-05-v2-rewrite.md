# Cascade: Drone Derby v2 — From-Scratch Rewrite

**Created:** 2026-07-05
**Status:** Phases 1–2 DONE — Phase 3 (Convex) next; Todd's fun-playtest pending

## Vision

Async multiplayer RoboRally-style programming game, browser-first PWA (no app
stores). Core loop: get a push notification → watch the last turn replay with
playback controls → program your 5 registers → submit → when the last player
submits, the turn executes and everyone gets notified. Games span days to a
week+.

## Cross-Phase Decisions (locked unless revisited)

- **Backend: Convex** (DB + auth + reactive queries + serverless functions).
  Replaces the entire old Express/Postgres/Redis/Socket.io stack. Pivot to
  Supabase possible later; keep Convex-specific code confined to `convex/`
  and thin client hooks.
- **Engine is a pure function**: `(gameState, programs, seed) → (newState,
  EventLog)`. Zero dependencies, deterministic (seeded RNG), lives in
  `src/engine/`, imported by BOTH the client (hot-seat, previews, replay) and
  Convex functions (authoritative execution). Never touches DB/DOM.
- **EventLog is the animation contract**: turn execution emits atomic events
  (move, rotate, push, conveyor, laser, damage, fall, respawn, checkpoint…)
  grouped by register. The replay player consumes only the EventLog. Stored
  per turn in Convex so any past turn is replayable.
- **Rendering: DOM/CSS grid** for the game board (tiles + transitions), NOT
  canvas. Canvas (Konva/Pixi) reserved for the future level editor only.
- **State: Zustand** for client state. No Redux. No MUI — plain CSS
  (custom properties, grid/flex); can add Tailwind later if wanted.
- **Auth: Convex Auth** (magic link + Google OAuth). No passwords, ever.
- **One app, one package.json**: Vite + React + TS at repo root, `convex/` for
  backend functions. No monorepo/workspaces.
- **Old codebase archived** at `legacy/` — reference only (card/tile ideas,
  editor UX). Never import from it.
- **Rules scope (MVP)**: RoboRally-lite. Cards: Move 1/2/3, Back Up, Rotate
  L/R, U-Turn, priority 10–840. Tiles: floor, walls (edge-based), pits,
  conveyors (normal + express), gears, checkpoints, spawn docks, board
  lasers. Robots fire lasers. Damage locks registers (5+) and kills at 10;
  3 lives; respawn at last checkpoint. Cut for MVP: pushers, repair sites,
  option cards, power-down.

### Phase 2 — Hot-Seat Game + Replay Player  [DONE 2026-07-05]
Delivered: Zustand store (`src/store/gameStore.ts`) — screen state machine
setup→handoff→programming→replay→gameover; turn seed = `initialSeed +
game.turn` so whole games replay from one number. DOM/CSS-grid board

captions, laser-beam + bump overlays. Programming UI
(`src/components/programming/`): hand → 5 registers (locked registers
shown with held card), handoff pass-the-device gate, setup + game-over
screens. Plain CSS dark theme in `src/index.css`. 58 tests green (4 new
reducer tests incl. full-game lockstep vs the engine on Proving Grounds).
Browser-verified via Playwright MCP (screenshots in `screengrab/`).
Notes for later phases:
- `robot-respawned` implies a damage reset (engine emits no damage event
  on respawn) — the visual reducer sets `RESPAWN_DAMAGE`; keep in sync.
- Robot facing animates via cumulative angles (shortest-path CSS spin).
